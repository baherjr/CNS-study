Computer & Network Security Overview

CIA Triad · OSI Architecture · Passive vs Active Attacks · Cryptographic Algorithms · Security Mechanisms & Services · Trust Model

1.1 — The CIA Triad

Confidentiality

Data confidentiality: Private info not disclosed to unauthorized parties.
Privacy: Individuals control what info is collected about them.

e.g. Encrypted WhatsApp messages
Integrity

Data integrity: Changes only in specified/authorized manner + authenticity + nonrepudiation.
System integrity: System performs intended function without unauthorized manipulation.

Availability

Systems work promptly and service is not denied to authorized users. Depends on access control and other security services.

Counters DoS attacks
⚠ Nonrepudiation falls under Integrity — not a standalone objective. Neither sender nor receiver can deny having processed the information.

1.7 — Security Services (X.800)

A security service enhances the security of data processing systems. Each service uses one or more security mechanisms and is intended to counter specific attacks.

🪪 Authentication

Concerned with assuring that a communication is authentic. Two specific authentication services are defined in X.800:

Peer Entity Authentication

Corroboration of the identity of a peer entity in an association. Used at connection establishment or during data transfer. Provides confidence that an entity is not performing a masquerade or unauthorized replay of a previous connection. Two entities are peers if they implement the same protocol in different systems.

Data Origin Authentication

Corroboration of the source of a data unit. Does not provide protection against duplication or modification. Supports applications like electronic mail where there are no ongoing interactions between communicating entities.

🚪 Access Control

The ability to limit and control the access to host systems and applications via communications links. To achieve this, each entity trying to gain access must first be identified or authenticated, so that access rights can be tailored to the individual.

Access control is a prerequisite for other services — authentication must succeed before access rights can be enforced.
🔒 Data Confidentiality

The protection of transmitted data from passive attacks. Two aspects:

Broadest Form

Protects all user data transmitted between two users over a period of time. Narrower forms protect a single message or specific fields within a message.

Traffic Flow Confidentiality

Protection against traffic analysis — an attacker must not be able to observe the source and destination, frequency, length, or other characteristics of traffic.

✅ Data Integrity

Can apply to a stream of messages, a single message, or selected fields within a message.

Connection-Oriented Integrity

Deals with a stream of messages. Assures messages are received as sent with no duplication, insertion, modification, reordering, or replays.

Connectionless Integrity

Deals with individual messages without regard to any larger context. Generally provides protection against message modification only.

🚫 Nonrepudiation

Prevents either sender or receiver from denying a transmitted message.

Proof of Origin

When a message is sent, the receiver can prove that the alleged sender in fact sent the message.

Proof of Delivery

When a message is received, the sender can prove that the alleged receiver in fact received the message.

⚠ Exam note: Nonrepudiation is classified under Integrity in the CIA Triad, but it is its own dedicated security service in the X.800 framework.
⚡ Availability of Service

Protects a system to ensure its availability. This service addresses the security concerns raised by denial-of-service (DoS) attacks.

It depends on proper management and control of system resources, and thus depends on access control service and other security services.

Availability is both a CIA objective (assure systems work promptly) and a dedicated X.800 security service that specifically counters DoS attacks.

▾

Nonrepudiation is classified under Integrity (specifically under data integrity). It means: the sender of information is provided with proof of delivery, and the recipient is provided with proof of the sender's identity — so neither can later deny having processed the information. Critical for legal and financial transactions.

CIA threat: primarily Confidentiality (specifically privacy — revealing who communicates with whom, even without reading content, violates informational privacy).

Introduction to Number Theory

2.1 — Divisibility

A nonzero b divides a (written b | a) if a = mb for some integer m — i.e. there is no remainder on division. We then say b is a divisor of a.

Examples: 13 | 182 · −5 | 30 · 17 | 289 · −3 | 33 · 17 | 0 Positive divisors of 24: 1, 2, 3, 4, 6, 8, 12, 24
Properties of Divisibility
PropertyStatementSlide Example
P1If a | 1, then a = ±1—
P2If a | b and b | a, then a = ±b—
P3Any b ≠ 0 divides 017 | 0 ✓
P4 TransitivityIf a | b and b | c, then a | c11|66 and 66|198 → 11|198
P5 Linear combinationIf b|g and b|h, then b|(mg+nh) for any integers m, nb=7, g=14, h=63, m=3, n=2: 7|(3·14+2·63)=7·24 ✓
⚠ Proof of P5: If b|g → g=b·g₁; if b|h → h=b·h₁; then mg+nh = b(mg₁+nh₁) → b divides mg+nh. ∎

2.2 — Division Algorithm

Given any positive integer n and any nonnegative integer a, dividing a by n yields a unique quotient q and remainder r:

a = qn + r     0 ≤ r < n     q = ⌊a/n⌋
POSITIVE EXAMPLE

11 mod 7 = 4
11 = 1×7 + 4  (0 ≤ 4 < 7 ✓)

NEGATIVE EXAMPLE (MCQ TRAP)

−11 mod 7 = 3 (NOT −4!)
−11 = (−2)×7 + 3  (0 ≤ 3 < 7 ✓)

⚠ Fig 2.1 example: 70 = (4 × 15) + 10 → q=4, r=10. The remainder is always non-negative even when a is negative.

2.3 — Greatest Common Divisor (GCD)

The greatest common divisor gcd(a,b) is the largest integer that divides both a and b. Positive integer c is the gcd if: (1) c divides both a and b, and (2) any divisor of both a and b also divides c.

Formal definition: gcd(a,b) = max{k : k|a and k|b} Key identities: gcd(a,b) = gcd(|a|,|b|) e.g. gcd(60,−24)=gcd(60,24)=12 gcd(a,0) = |a| (all nonzero integers divide 0) gcd(0,0) = 0 (by definition) gcd(a,b)=1 ⟺ a and b are relatively prime Example: 8 and 15 are relatively prime divisors of 8: {1,2,4,8} divisors of 15: {1,3,5,15} only common divisor = 1 → gcd(8,15)=1 ✓

2.4 — Euclidean Algorithm

Key GCD Recursion
gcd(a, b) = gcd(b, a mod b) gcd(a, 0) = a gcd(a, 1) = 1
C-style Pseudocode (from slides)
int gcd(int a, int b) { int r = a % b; if(r == 0) return b; if(r == 1) return 1; return gcd(b, r); }
Worked Example — gcd(710, 310)

Fig 2.3 from the slides:

710 = 2 × 310 + 90 → gcd(310, 90) 310 = 3 × 90 + 40 → gcd(90, 40) 90 = 2 × 40 + 10 → gcd(40, 10) 40 = 4 × 10 + 0 → gcd = 10
gcd(710, 310) = 10

Table 2.1 shows a larger example: gcd(1160718174, 316258250) = 1078 — computed in 10 iterations.

The Extended Euclidean Algorithm additionally finds integers x, y such that ax + by = gcd(a,b) — used to compute modular inverses in RSA key generation.

2.5 — Modular Arithmetic

If a is an integer and n is a positive integer, a mod n is the remainder when a is divided by n. The integer n is called the modulus.

a = ⌊a/n⌋ × n + (a mod n) → a mod n = a − ⌊a/n⌋ × n Examples: 11 mod 7 = 4 · −11 mod 7 = 3
Three Arithmetic Properties
[(a mod n) + (b mod n)] mod n = (a + b) mod n [(a mod n) − (b mod n)] mod n = (a − b) mod n [(a mod n) × (b mod n)] mod n = (a × b) mod n
Worked Examples (from slides, n=8)
11 mod 8 = 3 · 15 mod 8 = 7 [(11 mod 8) + (15 mod 8)] mod 8 = (3+7) mod 8 = 10 mod 8 = 2 (11 + 15) mod 8 = 26 mod 8 = 2 ✓ [(11 mod 8) − (15 mod 8)] mod 8 = (3−7) mod 8 = −4 mod 8 = 4 (11 − 15) mod 8 = −4 mod 8 = 4 ✓ [(11 mod 8) × (15 mod 8)] mod 8 = (3×7) mod 8 = 21 mod 8 = 5 (11 × 15) mod 8 = 165 mod 8 = 5 ✓

2.7 — Congruences

Two integers a and b are congruent modulo n if (a mod n) = (b mod n). Written: a ≡ b (mod n). Equivalently: n | (a − b).

73 ≡ 4 (mod 23) because 73−4=69=23×3 ✓ 21 ≡ −9 (mod 10) because 21−(−9)=30=10×3 ✓ 23 ≡ 8 (mod 5) because 23−8=15=5×3 ✓ −11 ≡ 5 (mod 8) because −11−5=−16=8×(−2) ✓ 81 ≡ 0 (mod 27) because 81−0=81=27×3 ✓ Note: if a ≡ 0 (mod n), then n | a
Three Properties of Congruences
  1. a ≡ b (mod n)  if n | (a − b)
  2. a ≡ b (mod n)  ⟹  b ≡ a (mod n)  (symmetry)
  3. a ≡ b (mod n) and b ≡ c (mod n)  ⟹  a ≡ c (mod n)  (transitivity)

Prove that [(a mod n) + (b mod n)] mod n = (a + b) mod n using the approach from the slides.

3.2 — Caesar Cipher

Simplest substitution cipher. Julius Caesar used a shift of 3. Each letter shifted k positions forward (wraps around).

Encrypt: C = E(k, p) = (p + k) mod 26 Decrypt: p = D(k, C) = (C − k) mod 26 Example (k=3): "meet me" → PHHW PH Only 25 possible keys → brute-force trivial
⚠ Only 25 keys — any attacker can try all shifts in seconds. Brute-force cryptanalysis is shown in Figure 3.3.

3.3 — Monoalphabetic Cipher

Strength

Key space = 26! ≈ 4×10²⁶ — 10 orders of magnitude larger than DES. Brute-force infeasible.

Weakness

Reflects the frequency distribution of the original language. Frequency analysis (using letter, digram, trigram frequencies) breaks it easily.

Most common English: E, T, A, O, I, N
Most common digram: th · Most common trigram: the

3.4 — Playfair Cipher

Best-known multiple-letter cipher. Encrypts digrams (pairs of letters) using a 5×5 keyword matrix. Invented by Charles Wheatstone in 1854. Used by British Army (WWI) and U.S. Army (WWII).

By encrypting pairs, Playfair defeats single-letter frequency analysis. There are 676 possible digrams vs only 26 letters.

3.5 — Polyalphabetic Ciphers & Vigenère

Uses multiple monoalphabetic substitutions as you progress through the plaintext. A key determines which substitution rule is applied at each position.

Vigenère Cipher — the best-known polyalphabetic cipher — uses a keyword to cycle through 26 Caesar ciphers (shifts 0–25).

Key: d e c e p t i v e d e c e p t i v e d e c Plain: t h i s i s a s e c r e t m e s s a g e Cipher: W L R W A L I N I F V E X J X E N E Z I Y
Same plaintext letter maps to different ciphertext letters — defeats frequency analysis. Still breakable via Kasiski test once key length is determined.

3.6 — Hill Cipher

Strength

Matrix multiplication completely hides single-letter frequencies. A 3×3 Hill cipher hides both single-letter and two-letter frequency information.

Weakness

Strong vs ciphertext-only attacks but easily broken with a known-plaintext attack — if attacker has plaintext-ciphertext pairs, they can solve for the key matrix.

3.8 — Product Ciphers & One-Time Pad

Product Ciphers

Single substitution or transposition ciphers are insecure due to language characteristics.

Sub + Sub → more complex substitution
Trans + Trans → more complex transposition
Sub + Trans → new, much harder cipher ✓
This is the bridge from classical to modern ciphers.
One-Time Pad
Why unbreakable

Random output with no statistical relation to plaintext. Only cryptosystem with perfect secrecy.

Two problems

1. Making large random keys
2. Mammoth key distribution — key length = message length

Decrypt: p = D(k, C) = (C − k) mod 26
k ∈ {1…25}. For k=3: 'meet me' → PHHW PH.

EasyMonoalphabetic
The monoalphabetic cipher has 26! possible keys. Why is it still insecure?
▾

Despite the enormous key space (26! ≈ 4×10²⁶), monoalphabetic ciphers are easy to break because they reflect the frequency distribution of the original language. Frequency analysis uses letter, digram (most common: 'th'), and trigram (most common: 'the') frequencies to crack the substitution.

EasyPlayfair
What makes the Playfair cipher stronger than a simple monoalphabetic cipher?
▾

Playfair encrypts digrams (pairs of letters) rather than single letters. This defeats single-letter frequency analysis since there are 676 possible digrams vs only 26 letters. It was invented by Wheatstone in 1854 and used by U.S. Army and Allied forces in WWII.

EasyTransposition
What is the simplest transposition cipher? Describe how it works.
▾

The Rail Fence cipher is the simplest transposition cipher. Plaintext is written diagonally across "rails" and then read off row by row. Example: "meet me after the toga party" with depth 2 → rows MEMATRHTGPRY and ETEFETEOAAT → ciphertext MEMATRHTGPRYETEFETEOAAT.

EasyOne-Time Pad
Why is the one-time pad theoretically unbreakable? What are its two practical difficulties?
▾

Unbreakable because it produces random output with no statistical relationship to the plaintext. The ciphertext contains zero information about the plaintext → perfect secrecy (only cryptosystem with this property).

Two difficulties: 1. Making large quantities of truly random keys. 2. Mammoth key distribution problem — each message needs a key of equal length securely delivered.

MediumPolyalphabetic
What two features do ALL polyalphabetic ciphers share? How does Vigenère implement these?
▾

All polyalphabetic ciphers: (1) use a set of related monoalphabetic substitution rules, (2) a key determines which rule is chosen for each position.

Vigenère uses all 26 Caesar ciphers (shifts 0–25) as its rule set. The keyword cycles to determine which shift applies at each plaintext position — same letter gets different ciphertext depending on position.

MediumHill Cipher
Against which type of attack is the Hill cipher strong, and against which is it weak?
▾

Strong against: ciphertext-only attacks — matrix multiplication completely hides single-letter (and with 3×3 matrix, also two-letter) frequency information.

Weak against: known-plaintext attacks — if an attacker has plaintext-ciphertext pairs, they can set up a matrix equation to directly solve for the key matrix.

MediumRow Transposition
In the Row Transposition cipher, what serves as the key?
▾

The order of the columns serves as the key. The message is written in a rectangle row by row, then read off column by column — but the columns are read in the order specified by the key. Example: key 4312567 determines the column reading order.

HardProduct Ciphers
Why does Sub + Trans produce a much harder cipher than Sub + Sub or Trans + Trans?
▾

Sub + Sub = a more complex substitution (still a substitution, same weakness to frequency analysis).
Trans + Trans = a more complex transposition (still a transposition, same weakness to pattern analysis).

Sub + Trans = an entirely new type of cipher that doesn't reduce to either alone. The combined statistical properties are far harder to exploit. This principle — combining different cipher types — is the bridge from classical to modern ciphers, and is the basis of AES and DES which combine multiple rounds of substitutions and transpositions (permutations).

HardScenario
You intercept a ciphertext and do frequency analysis. The top 3 letters by frequency are P (13.3%), Z (11.7%), S (8.3%). What type of cipher is most likely being used, and how do you proceed?
▾

The clear peak in letter frequencies (P at 13.3%, Z at 11.7%) mirrors English frequency distribution — this strongly suggests a monoalphabetic substitution cipher (polyalphabetic ciphers tend to flatten the distribution).

Proceed: Map P→E (most common English letter) and Z→T (second most common). Substitute these throughout and look for recognizable patterns. Use digram analysis ('th' → look for PZ or ZP patterns) to confirm and extend the mapping until the full alphabet is recovered. This is the exact example from the slides.

4.1 — Stream Cipher vs Block Cipher

Stream Cipher

Encrypts a digital data stream one bit or one byte at a time. Uses a keystream generator — each plaintext bit is XORed with a keystream bit.

Autokeyed Vigenère Vernam cipher
If keystream is truly random → equivalent to one-time pad → unbreakable. Practical keystreams are pseudo-random.
Block Cipher

A block of plaintext is treated as a whole and used to produce a ciphertext block of equal length. Both users share a symmetric encryption key.

Typical block sizes: 64 or 128 bits Y = E(K, X) → X = D(K, Y) Majority of network symmetric crypto applications use block ciphers

4.2 — Diffusion & Confusion (Shannon)

Terms introduced by Claude Shannon to capture the two basic building blocks for any cryptographic system. Shannon's concern: thwart cryptanalysis based on statistical analysis.

Diffusion

The statistical structure of the plaintext is dissipated into long-range statistics of the ciphertext. Achieved by having each plaintext digit affect the value of many ciphertext digits.

Implemented by: permutations
Confusion

Makes the relationship between ciphertext statistics and encryption key as complex as possible. Even if attacker handles ciphertext statistics, deducing the key remains infeasible.

Implemented by: substitutions

4.3 — Feistel Cipher

Proposed by Horst Feistel — a practical application of Shannon's product cipher proposal. Alternates substitutions and permutations. The structure used by many significant symmetric block ciphers currently in use.

For each round i (encryption): L_i = R_{i-1} R_i = L_{i-1} ⊕ F(R_{i-1}, K_i) Decryption is the INVERSE — same algorithm, subkeys applied in REVERSE order (K_16 → K_1)
Key insight: Decryption uses the exact same hardware/software as encryption — only the key schedule is reversed. This makes Feistel very efficient to implement.

4.7 — Avalanche Effect

A desirable property of any encryption algorithm: a small change in either the plaintext or key should produce a significant change in the ciphertext. Specifically, changing one bit in the input should change approximately half the output bits.

CHANGE IN PLAINTEXT (Table 4.3)

Changing 1 plaintext bit → after all 16 rounds, approximately 30+ bits differ in the ciphertext (δ ≈ 30). Effect grows with each round.

CHANGE IN KEY (Table 4.4)

Changing 1 key bit → after all 16 rounds, approximately 30 bits differ in the ciphertext (δ ≈ 30). Effect propagates quickly from round 1.

The avalanche effect ensures that attackers cannot infer any information about the key or plaintext from small changes in the ciphertext — critical for security.

EasyShannon
Who introduced diffusion and confusion? Define each term.

EasyFeistel
State the two equations for a Feistel encryption round.
▾

For round i:
L_i = R_{i-1}
R_i = L_{i-1} ⊕ F(R_{i-1}, K_i)

The left half of the new output is simply the previous right half. The new right half XORs the previous left half with the output of the round function F applied to the previous right half and the subkey.

EasyAvalanche
Define the avalanche effect. Why is it desirable?
▾

The avalanche effect: a small change in either the plaintext or key should produce a significant change in the ciphertext. Ideally, changing 1 bit should change approximately half the output bits.

It is desirable because it ensures attackers cannot infer any information about the key or plaintext by making small changes and observing the ciphertext differences.

MediumFeistel Decryption
How does Feistel decryption work? What is the key property that makes it elegant?
▾

Feistel decryption uses the exact same algorithm as encryption — the only difference is that the subkeys are applied in reverse order (K_16 first, K_1 last for a 16-round cipher).

To reverse round i: XR = R_i ⊕ F(L_i, K_i) = L_{i-1}. This works because XORing with F twice cancels it out (A ⊕ B ⊕ B = A).

The elegance: the same hardware or software circuit can be used for both encryption and decryption — only the key schedule input needs to change.

MediumDES History
What was the DES design controversy? What was the outcome?
▾

Two main concerns:
1. The choice of a 56-bit key (vs Lucifer's 128-bit) — seen as too small, possibly to allow government brute-force.
2. The design criteria for the S-boxes were classified — raising suspicion of a backdoor.

Outcome: Subsequent public analysis showed the design was appropriate. The S-box structure was later found to provide resistance against differential cryptanalysis (a technique discovered publicly years later). DES flourished, especially in financial applications.

MediumKey Search Table
According to Table 4.5, how long does it take to crack DES at 10¹³ decryptions/second? How does this compare to AES-128?
▾

DES (56-bit): At 10¹³ dec/s → 1 hour. Completely broken at modern computational speeds.

AES-128: At 10¹³ dec/s → 5.3 × 10¹⁷ years. Effectively unbreakable by exhaustive search.

Triple DES (168-bit): 5.8 × 10²⁹ years — between DES and AES in the comparison. The exponential growth of the key space means each extra bit doubles the search time.

HardFeistel Design
For each of the 6 Feistel design parameters, explain the trade-off between security and performance.
▾

Block size: Larger → greater security (more plaintext per block, harder to attack) but slower enc/dec for a given algorithm complexity.

Key size: Larger → greater security (exponentially more key candidates) but may slow down key scheduling and operations. 64-bit now inadequate; 128-bit is current standard minimum.

Number of rounds: More → greater security (target: breaking cipher no easier than exhaustive search) but proportionally slower.

Subkey generation: Greater complexity → harder to work back to master key, but adds computational overhead per round.

Round function F: More complex → greater resistance, but each round takes longer. S-boxes provide non-linearity essential for confusion.

Fast software enc/dec: When hardware is not available, algorithm speed in software matters for practical deployment — simpler operations (XOR, shifts) are preferred.

HardScenario
An engineer proposes a block cipher with 32-bit blocks, 32-bit key, and 4 rounds for use in a payment system. Evaluate this proposal using the Feistel design principles.
▾

Block size (32-bit) — INSUFFICIENT: Small blocks enable codebook attacks — with a 32-bit block, an attacker who observes 2³² ciphertext blocks (feasible for a payment system) can build a complete codebook mapping all possible plaintexts.

Key size (32-bit) — CRITICALLY WEAK: 2³² ≈ 4 billion keys. At 10⁹ dec/s this takes about 4 seconds. At modern speeds it's broken instantly. By Table 4.5 logic, we need at minimum 128 bits.

4 rounds — INSUFFICIENT: The Feistel principle states a single round offers inadequate security; the criterion is that breaking the cipher should require effort no less than exhaustive key search. 4 rounds is trivially short — DES uses 16.

Verdict: This cipher is cryptographically unacceptable for any real application. Minimum recommendation: 128-bit block, 128-bit key, 10+ rounds — i.e., use AES.

5.1 — Groups

A group is a set G with a binary operation • satisfying the following axioms for all a, b, c ∈ G:

AxiomNameStatement
A1ClosureIf a, b ∈ G then a • b ∈ G
A2Associativitya • (b • c) = (a • b) • c
A3Identity∃ e ∈ G such that a • e = e • a = a
A4Inverse∀ a ∈ G, ∃ a⁻¹ ∈ G such that a • a⁻¹ = e
A5Commutativea • b = b • a  (makes it an Abelian group)
A group with axioms A1–A4 only is a plain group. Adding A5 (commutativity) gives an Abelian group.
Cyclic Groups

A group G is cyclic if every element of G is a power aᵏ (k an integer) of a fixed element a ∈ G. That element a is called a generator of G.

a⁰ = e (identity element) a⁻ⁿ = (a')ⁿ where a' is the inverse of a A cyclic group is always ABELIAN and may be finite or infinite

5.2 — Rings

A ring {R, +, ×} is an Abelian group under addition (axioms A1–A5) plus additional axioms for multiplication:

AxiomNameStatement
M1Closure (×)If a, b ∈ R then ab ∈ R
M2Associativity (×)a(bc) = (ab)c
M3Distributivea(b+c) = ab+ac  and  (a+b)c = ac+bc
M4Commutativity (×)ab = ba  → Commutative ring
M5Multiplicative identity∃ 1 such that a·1 = 1·a = a  → adds Integral domain
M6No zero divisorsab = 0 ⟹ a = 0 or b = 0
Ring = Abelian group + M1–M3. Commutative ring adds M4. Integral domain adds M5–M6.

5.3 — Fields

A field {F, +, ×} is an integral domain (A1–M6) plus:

AxiomStatement
M7 Multiplicative inverse∀ a ∈ F, a ≠ 0, ∃ a⁻¹ ∈ F such that aa⁻¹ = 1

A field is a set where we can do addition, subtraction, multiplication, and division without leaving the set. Division: a/b = a(b⁻¹).

Examples of fields

Rational numbers ℚ
Real numbers ℝ
Complex numbers ℂ
GF(p) — integers mod prime p

NOT a field

The integers ℤ — because not every element has a multiplicative inverse (e.g., 2⁻¹ ∉ ℤ).

5.4 — Finite Fields: GF(p)

The order of a finite field must be a power of a prime pⁿ (n ≥ 1). GF(pⁿ) = Galois field of order pⁿ, named after Évariste Galois (1811–1832).

GF(p) — Properties
  1. Consists of p elements
  2. Addition and multiplication defined over the set
  3. All four arithmetic operations possible without leaving the set
  4. Each non-zero element has a multiplicative inverse
Construction

Elements of GF(p) = integers {0, 1, …, p−1}. Arithmetic operations = addition and multiplication mod p.

Example: GF(7) = {0,1,2,3,4,5,6} 3 + 5 = 8 mod 7 = 1 3 × 5 = 15 mod 7 = 1 ∴ 3 and 5 are multiplicative inverses in GF(7)

5.5 — Polynomial Arithmetic over GF(2)

Polynomials with binary (0 or 1) coefficients. Operations are performed mod 2 (coefficient arithmetic) and, for GF(2ⁿ), also mod an irreducible polynomial of degree n.

Addition / Subtraction

Coefficient addition mod 2 = XOR. In GF(2), addition and subtraction are identical.

(x⁷+x⁵+x⁴+x³+x+1) + (x⁷+x³+x) = x⁵+x⁴+1
Multiplication

Ordinary polynomial multiplication, then reduce coefficients mod 2 (XOR). For GF(2ⁿ): also reduce mod irreducible polynomial.

Implemented by: shift and XOR
Modular Reduction

Repeatedly substitute highest power with remainder of irreducible polynomial (shift and XOR).

AES uses: x⁸+x⁴+x³+x+1

5.6 — GF(2ⁿ) and Two Uses of mod

GF(2ⁿ) contains 2ⁿ elements, each represented as an n-bit string (polynomial with binary coefficients of degree ≤ n−1).

Use 1 of mod

mod p on integers — reduces coefficients to {0,1,…,p−1}. In GF(2ⁿ), this is mod 2, giving binary coefficients only.

Use 2 of mod

mod p(x) on polynomials — reduces the result polynomial by an irreducible polynomial, keeping the degree below n.

Why GF(2⁸) matters for AES: AES operates on bytes (8 bits) as elements of GF(2⁸). Each byte is a polynomial in GF(2⁸), and the S-box is defined as a multiplicative inverse in GF(2⁸) mod (x⁸+x⁴+x³+x+1).

MediumTwo Uses of mod
Explain the two different uses of the mod operator in GF(2ⁿ).

What is a cyclic group and what is a generator?

A group G is cyclic if every element of G is a power aᵏ (k an integer) of a fixed element a ∈ G. That element a is called a generator of G.

Key properties: every cyclic group is Abelian; it may be finite or infinite. In a finite field GF(pⁿ), a root of an irreducible polynomial is a generator — its powers enumerate all nonzero elements, as shown in Table 5.5 for GF(2³) with generator satisfying x³+x+1=0.

HardGF(2³)
Using GF(2³) with irreducible polynomial x³+x+1, compute g³ and explain why using the generator table.
▾

From Table 5.5: g³ = g + 1 (binary 011, decimal 3).

Derivation: g is defined as a root of x³+x+1=0, so g³ = −g−1 = g+1 (in GF(2), −1 = 1, so subtraction = addition).

HardApplication
Why do cryptographic algorithms like AES operate in GF(2⁸) rather than ordinary integer arithmetic?
▾

AES processes bytes (8-bit values), so it needs a mathematical structure over 8-bit strings. GF(2⁸) provides this because:

1. Closure — all operations stay within 8-bit values (no overflow issues).
2. Multiplicative inverses exist — every nonzero byte has an inverse, enabling the S-box substitution (critical for confusion).
3. Efficient hardware implementation — GF(2⁸) operations reduce to XOR and shifts, which are fast in hardware and software.
4. Well-defined structure — the algebraic properties guarantee security-relevant behavior like the avalanche effect when implemented correctly.

If ordinary integer arithmetic were used, overflow management would break the bijective property of the S-box, and multiplicative inverses wouldn't be guaranteed.
