Structured Summary

1 · Cybersecurity — Three Branches

BranchFocus
CybersecurityAttainment and maintenance of security properties against risks in cyberspace
Information SecurityPreservation of confidentiality, integrity, and availability (CIA) of information
Network SecurityProtection of networks from unauthorized modification, destruction, or disclosure

2 · The CIA Triad — Core Security Objectives

C
Confidentiality
Data confidentiality: Private info not disclosed to unauthorized parties.

Privacy: Individuals control what info is collected and by whom.
e.g., Encrypted WhatsApp messages
I
Integrity
Data integrity: Changes only in specified/authorized manner + authenticity + nonrepudiation.

System integrity: System performs intended function without unauthorized manipulation.
e.g., Detecting tampered bank records
A
Availability
Assures that systems work promptly and service is not denied to authorized users.

Depends on access control and other security services.
e.g., Resisting a DDoS attack
⚠ Exam Alert Nonrepudiation is listed under Integrity — not as a separate objective. Neither sender nor receiver can deny having processed the information.

3 · OSI Security Architecture — 3 Core Terms

TermDefinition
Security AttackAny action that compromises the security of information owned by an organization
Security MechanismA process designed to detect, prevent, or recover from a security attack
Security ServiceEnhances security of data processing; uses one or more mechanisms to counter attacks
Security Services include: Authentication Access Control Data Confidentiality Data Integrity Nonrepudiation Availability

4 · Security Attacks — Taxonomy

Passive Attacks
Hard to detect · Goal = obtain information

Eavesdropping or monitoring transmissions without affecting system resources.

Release of message contents Traffic analysis
Active Attacks
Hard to prevent · Goal = alter or disrupt

Modification of data stream or creation of a false stream.

Masquerade Replay Data Modification Denial of Service
Active Attack TypeDefinition
MasqueradeOne entity pretends to be a different entity; usually includes another form of active attack
ReplayPassive capture of a data unit and its subsequent retransmission to produce an unauthorized effect
Data ModificationLegitimate message is altered, or messages are delayed/reordered
Denial of ServicePrevents or inhibits normal use or management of communications facilities
🔑 Key Exam Distinction Passive attacks are hard to detect. Active attacks are hard to prevent — the goal is to detect them and recover.

Key exchange

MAC: Uses a secret key + hash function to verify integrity. Recipient recalculates MAC — if it matches, the message is unaltered.

Digital Signature: Computed value that lets any recipient verify data origin and integrity.

6 · Trust Model

Trust is the "willingness of a party to be vulnerable to the actions of another party based on the expectation that the other will perform a particular action important to the trustor."

🏅
Trustworthiness
Characteristic reflecting the degree to which an entity deserves trust
🤝
Propensity to Trust
Baseline willingness to rely on words and actions of others across situations
⚖️
Risk
f(adverse impact × likelihood of occurrence) — measure of threat to an entity

Two factors for trustworthy systems (NIST SP 800-39): Security functionality (crypto/network features) + Security assurance (confidence they work, via auditing & SDLC).

7 · Security Challenges

Security is not simple; attacks must be anticipated in advance
Procedures used are often counter-intuitive
Security is too often an afterthought
Essentially a battle of wits between attacker and designer
Strong security viewed as an impediment to usability
Little benefit perceived until a security failure occurs
Requires constant monitoring
Security mechanisms involve more than a single algorithm
